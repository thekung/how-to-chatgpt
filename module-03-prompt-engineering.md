# Module 3 : วิธีเขียน Prompt อย่างถูกต้อง (Prompt Engineering)

# Prompt คืออะไร

Prompt คือ "คำสั่ง"

> "คำสั่ง หรือข้อความที่ผู้ใช้ป้อนให้ AI เพื่อบอกว่า ต้องการให้ AI ทำอะไร"

Prompt เปรียบเสมือนการสั่งงานพนักงานใหม่ ถ้าสั่งงานชัดเจนผลลัพธ์ก็จะมีคุณภาพ แต่ถ้าสั่งงานไม่ชัดเจน AI ก็ต้อง "เดา" สิ่งที่ผู้ใช้ต้องการ

---

# ตัวอย่าง Prompt

Prompt ที่ไม่ดี

```
เขียนรายงาน
```

AI ไม่รู้ว่า

- รายงานเรื่องอะไร
- สำหรับใคร
- ความยาวเท่าไร
- ใช้ภาษาแบบไหน

---

Prompt ที่ดี

```
ช่วยเขียนรายงาน

หัวข้อ : การประยุกต์ใช้ AI ในองค์กร

สำหรับผู้บริหาร

ความยาวประมาณ 2 หน้า

ใช้ภาษาทางการ

พร้อมสรุป Executive Summary
```

ผลลัพธ์จะมีคุณภาพมากกว่า

---

# หลักการ Prompt Engineering

Prompt Engineering

คือ

> ศิลปะและเทคนิคในการเขียนคำสั่งให้ AI เข้าใจความต้องการของเราได้ดีที่สุด

เป้าหมายคือ

- ได้คำตอบที่ตรงความต้องการ
- ลดการถามซ้ำ
- ลดเวลาแก้ไข
- เพิ่มคุณภาพของผลลัพธ์

---

# หลักการสำคัญ

Prompt ที่ดีควรตอบคำถามต่อไปนี้

- ใคร (Role)
- ทำอะไร (Task)
- เพื่ออะไร (Objective)
- มีข้อมูลอะไร (Context)
- ต้องการผลลัพธ์แบบไหน (Output)
- มีข้อกำหนดอะไร (Constraint)

---

# Framework : R-T-C-O-O-C

```
Role
Task
Context
Objective
Output
Constraint
```

จำง่าย ๆ

> "บอก AI ว่าเป็นใคร ทำอะไร มีข้อมูลอะไร ต้องการอะไร และมีข้อจำกัดอย่างไร"

---

# 1. Role

Role

คือ

กำหนดบทบาทให้ AI

ตัวอย่าง

```
คุณคือผู้เชี่ยวชาญ Linux

```

หรือ

```
คุณคืออาจารย์มหาวิทยาลัย

```

หรือ

```
คุณคือผู้เชี่ยวชาญด้านการตลาด

```

---

## ทำไมต้องกำหนด Role

Role จะช่วยให้ AI

- ใช้ภาษาที่เหมาะสม
- เลือกตัวอย่างที่เหมาะสม
- ปรับระดับความลึกของคำตอบ

---

## ตัวอย่าง

ไม่มี Role

```
อธิบาย Docker
```

มี Role

```
คุณคือวิศวกรระบบ Linux

อธิบาย Docker
```

---

# 2. Task

Task

คือ

สิ่งที่ต้องการให้ AI ทำ

เช่น

- เขียน
- สรุป
- วิเคราะห์
- เปรียบเทียบ
- แปล
- ตรวจสอบ
- สร้าง

---

ตัวอย่าง

```
ช่วยสรุปรายงานนี้
```

```
ช่วยวิเคราะห์ SWOT
```

```
ช่วยเปรียบเทียบ VMware กับ KVM
```

---

# 3. Context

Context

คือ

ข้อมูลประกอบ

ยิ่งมี Context มาก

AI ยิ่งตอบตรงความต้องการ

---

ตัวอย่าง

```
บริษัทของฉัน

มีพนักงาน 300 คน

กำลังวางระบบ Virtualization

ต้องการลดค่าใช้จ่าย
```

AI จะตอบแตกต่างจาก

```
ช่วยแนะนำ Virtualization
```

---

# 4. Objective

Objective

คือ

เป้าหมายของงาน

ตัวอย่าง

```
เพื่อเสนอต่อผู้บริหาร
```

หรือ

```
เพื่อใช้ในการอบรม
```

หรือ

```
เพื่อใช้ในคู่มือ
```

---

# 5. Output

ระบุรูปแบบผลลัพธ์

เช่น

- ตาราง
- Markdown
- Word
- PowerPoint
- JSON
- CSV
- Bullet
- Checklist

---

ตัวอย่าง

```
สรุปเป็นตาราง
```

```
ตอบเป็น Markdown
```

```
ทำเป็น Checklist
```

---

# 6. Constraint

Constraint

คือ

ข้อกำหนดเพิ่มเติม

เช่น

- ไม่เกิน 500 คำ
- ใช้ภาษาทางการ
- ใช้ Bullet
- ห้ามใช้ศัพท์เทคนิค
- ยกตัวอย่าง 5 ตัวอย่าง

---

ตัวอย่าง

```
ตอบไม่เกิน 300 คำ

ใช้ Bullet

พร้อมตัวอย่าง 3 ตัวอย่าง
```

---

# Chain of Tasks

งานใหญ่

ควรแบ่งเป็นหลาย Prompt

ตัวอย่าง

ไม่ควรถาม

```
ช่วยสร้างธุรกิจใหม่
```

แต่ควรแบ่งเป็น

ขั้นที่ 1

```
ช่วยคิด Business Idea
```

ขั้นที่ 2

```
ช่วยวิเคราะห์ SWOT
```

ขั้นที่ 3

```
ช่วยเขียน Business Plan
```

ขั้นที่ 4

```
ช่วยสร้าง Presentation
```

ผลลัพธ์จะดีกว่า
